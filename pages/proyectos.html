1. Tour del caballo

Contexto

De que trata el tour del caballo? Sabemos que en el juego de ajedrez, el caballo es una pieza especial en terminos de movilidad: puede moverse en L y saltar piezas, pero para este proyecto nos concentramos en el movimiento.

Es posible que el caballo recorra todo el tablero sin repetir casilla?

Para este proyecto fueron necesarios los siguientes conocimientos:

  • Conocimiento en Java
  • Estructuras de datos y arreglos
  • Backtracking, muy usado en este tipo de problemas de permutaciones

En sintesis, el caballo tenia unas restricciones de movimiento (tanto suyas como del tablero) y un objetivo final, esa es la esencia del backtracking. Se toman caminos, y dependiendo de las restricciones, se debe volver atras y empezar de nuevo.

2. Compresion de Huffman

Contexto

Sabemos que un computador almacena informacion de manera binaria, por lo que normalmente se necesitan varias reglas y digitos para poder detectar cuando empieza y cuando acaba otro caracter. Sin embargo, estos digitos extra aumentan la cantidad de memoria que ocupa un solo texto, por lo que la compresion de Huffman usa una solucion ingeniosa a ese problema.

Se puede crear un metodo mas eficiente de comprimir un archivo de texto para que no ocupe tanta memoria?

Para este proyecto, que tambien fue hecho en clase de Estructuras de datos, usamos:

  • Conocimiento en Java
  • Estructuras de datos, tales como pilas y colas
  • Arboles binarios, para poder hacer la secuencia de compresion
  • Creacion, modificacion y conocimiento de archivos y sus propiedades

Aunque solo sirve para textos, esta compresion sirve mucho para alivianar la memoria del computador. El archivo de texto se convierte en caracteres, y usando un arbol binario, se crea una secuencia para cada caracter. Asi, no se requieren cadenas largas solo para un caracter, y se tiene acceso a la clave para cuando se vaya a descomprimir.
